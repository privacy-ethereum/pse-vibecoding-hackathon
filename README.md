# PSE Vibecoding Hackathon

A 2-day remote hackathon for PSE members. Build whatever you want — vibecoding encouraged.

## Key Dates

| What | When |
|------|------|
| Kickoff (live) | Mon Mar 9, 2026 — 13:00 UTC |
| Submission deadline | Tue Mar 10, 2026 — 12:00 UTC |
| Demo & Judging (live) | Tue Mar 10, 2026 — 13:00 UTC |

## Rules

- **Eligibility**: PSE members only.
- **Team size**: 2–4 recommended, solo is fine.
- **What to build**: Anything. PSE process improvements, internal tooling, dogfooding our own tech, something completely unhinged. The only criteria: it should work, and you should be able to explain why it matters.
- **Vibecoding**: Fully encouraged. Use AI tools however you want — Claude, Copilot, Cursor, whatever. No restrictions on tooling.
- **Submissions**: Must be in this repo before the demo session (Mar 10, 12:00 UTC).
- **Demos**: Each team gets 5 minutes to present + 2 minutes Q&A during the live judging session.

## Prizes

| Place | Prize |
|-------|-------|
| 1st | $1,200 |
| 2nd | $500 |
| 3rd | $300 |

## Judging Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| Usefulness | 30% | Does this solve a real problem? Would people actually use it? |
| Working demo | 25% | Does it work? Can you show it running? |
| Creativity | 20% | Is the idea novel or surprising? |
| Implementation quality | 15% | Is the code reasonable? Could someone pick this up after the hackathon? |
| Presentation | 10% | Was the demo clear and compelling? |

## How to Submit

1. Create a folder: `submissions/your-team-name/`
2. Add your code and a `README.md` using the template below
3. Push directly to `main` or open a PR — either works

### Submission README Template

```markdown
# [Project Name]

## Team
- [Name 1]
- [Name 2]

## Problem
What workflow or problem does this address?

## Solution
What did you build? How does it work?

## Demo
Link to recording, screenshots, or instructions to run it live.

## How to Run
Steps to set up and run the project locally.

## Impact
What's the expected measurable outcome if this gets adopted?

## What's Next
Could this be maintained after the hackathon? What would a 30-day plan look like?
```
